User-space environments the fast way

Get started

Quick

Downloading a guest image from scratch and setting up a new environment takes a single command:

$ dockie run voidlinux:current /bin/bash

root@voidlinux #

Remote Images

Dockie allows pulling images from the LXC image server and Docker Hub with thousands of images available.

Unprivileged

With PRoot and Dockie, users don't need root privileges to use a guest filesystem or executing programs built for a different architecture. This makes possible to have a small guest running on top of a host for which root privileges are not availble, with Android + Termux being an example.

Lightweight

Dockie is made out of portable shell scripts and only needs PRoot, a user-space implementation of chroot and other utilities, to work.

Simple

Dockie provides an easy workflow to manage different filesystems, making it easier create, run and destroy different environments.
